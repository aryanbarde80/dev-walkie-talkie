# Documentation Guide 📖

Welcome to the Dev Walkie-Talkie documentation. This directory is structured to provide deep insights into the technical implementation and maintenance of the project.

## Directory Map

-   [**ARCHITECTURE.md**](./ARCHITECTURE.md): Start here to understand the system design.
-   [**WEBRTC_FLOW.md**](./WEBRTC_FLOW.md): Detailed logic of the signaling and streaming process.
-   [**PWA_FEATURES.md**](./PWA_FEATURES.md): Information regarding installability and offline support.
-   [**SETUP_GUIDE.md**](./SETUP_GUIDE.md): Technical requirements and environment configuration.

## Writing Style
Documentation should follow these principles:
1.  **Technical Accuracy**: Always verify code paths before documenting.
2.  **Visual Aids**: Use Mermaid diagrams or Markdown tables where possible.
3.  **Clarity**: Explain "why" a design choice was made (e.g., why we use `enabled = false` instead of `stop()`).

## Contributing to Docs
If you find a gap in the documentation, please submit a PR with the updated `.md` file in this directory.
